## IPv6 Address Configuration – Complete Summary

When a **PC joins an IPv6 network**, it automatically configures its network settings using:

* **SLAAC (Stateless Address Auto-Configuration)**
* **ICMPv6 messages (RS & RA)**
* **NDP (Neighbour Discovery Protocol)**
* **DHCPv6 (for DNS information)**

***

## 1. Link-Local Address Creation (LLA)

As soon as a device connects to a network, it **automatically creates a Link-Local Address**.

Example:

```
fe80::52:911c:a370:b03f
```

Rules:

* All LLAs start with **fe80::/10**
* Generated automatically by the host
* Used **only inside the local LAN**

Purpose:

* Communication with the router
* Neighbor discovery
* Address configuration

***

## 2. Router Solicitation (RS)

The PC wants to discover routers on the network.

It sends a **Router Solicitation (ICMPv6)** message.

Destination address:

```
ff02::2
```

Meaning:

> **All routers on the local network**

#### Diagram

```
PC
 |
 |  Router Solicitation (RS)
 |---------------------------->
 |        ff02::2
 |
Router
```

***

## 3. Router Advertisement (RA)

The router responds with a **Router Advertisement (ICMPv6)**.

Destination address:

```
ff02::1
```

Meaning:

> **All nodes on the local network**

The router sends configuration information.

#### Diagram

```
Router
 |
 | Router Advertisement (RA)
 |---------------------------->
 |        ff02::1
 |
PC
```

***

## 4. Information Inside the RA Message

The RA includes important network configuration:

| Information        | Purpose                   |
| ------------------ | ------------------------- |
| Network prefix     | Identifies the network    |
| Prefix length      | Defines subnet size       |
| Router MAC address | Link-layer option         |
| MTU                | Maximum transmission unit |
| Default gateway    | Router LLA (e.g. fe80::1) |
| DHCPv6 flag        | Tells host to use DHCPv6  |

Example prefix:

```
2001:db8:1:2::/64
```

***

## 5. Global Unicast Address (GUA) Creation

Using **SLAAC**, the PC creates its global IPv6 address.

Formula:

```
Network Prefix + Interface ID
```

Example:

```
Prefix:       2001:db8:1:2::/64
Interface ID: abcd:1234:5678:9abc
```

Final IPv6 address:

```
2001:db8:1:2:abcd:1234:5678:9abc
```

***

## 6. Duplicate Address Detection (DAD)

Before using the new IPv6 address, the PC checks that **no other device is using it**.

Protocol used:

**NDP (Neighbour Discovery Protocol)**

Process:

1. PC sends a Neighbor Solicitation
2. If **no reply** is received → address is unique

Important note:

> **NDP replaces ARP in IPv6 networks.**

***

## 7. DHCPv6 for DNS Information

Sometimes the Router Advertisement tells the PC:

> “Use DHCPv6 to obtain DNS server information.”

So the PC sends a DHCPv6 discover message.

Destination multicast address:

```
ff02::1:2
```

Meaning:

> **All DHCPv6 servers on the network**

#### Diagram

```
PC
 |
 | DHCPv6 Discover
 |---------------------------->
 |        ff02::1:2
 |
DHCPv6 Server
```

***

## 8. DHCPv6 Exchange

The DHCPv6 process follows four steps.

#### Step 1 – Discover

```
PC → ff02::1:2
```

Find DHCPv6 servers.

#### Step 2 – Advertise

```
Server → PC
```

Server announces it can provide configuration.

#### Step 3 – Request

```
PC → Server
```

PC requests configuration.

#### Step 4 – Reply

Server sends configuration information.

Example DNS server:

```
2001:db8:1:2::254
```

***

## 9. Final Configuration

After the process completes, the PC has the following information.

| Configuration       | Source            |
| ------------------- | ----------------- |
| Link-Local Address  | Generated by host |
| Global IPv6 Address | SLAAC             |
| Default Gateway     | Router LLA        |
| DNS Server          | DHCPv6            |

## Important Multicast Addresses to Remember

| Address     | Meaning            |
| ----------- | ------------------ |
| `ff02::1`   | All nodes          |
| `ff02::2`   | All routers        |
| `ff02::1:2` | All DHCPv6 servers |

***

## Key Protocols

| Protocol  | Function                                  |
| --------- | ----------------------------------------- |
| SLAAC     | Automatic IPv6 address creation           |
| ICMPv6 RS | Host discovers routers                    |
| ICMPv6 RA | Router provides network info              |
| NDP       | Neighbor discovery & duplicate detection  |
| DHCPv6    | Provides DNS and additional configuration |
