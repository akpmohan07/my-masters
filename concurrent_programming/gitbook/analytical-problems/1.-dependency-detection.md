***



**Fissioning ->**&#x20;



**Two operations in the loop:**

```c
data[i] = data[i] + data[i-1];   // line 1
result  = result + temp[i];       // line 2
```

***

**Line 1 — data dependency:**

```
data[i] depends on data[i-1]  → RAW dependency
i=1 needs data[0]
i=2 needs data[1]
i=3 needs data[2]
→ cannot parallelise — must stay sequential
```

***

**Line 2 — no dependency:**

```
result += temp[i]
each temp[i] is independent
→ safe to parallelise → use reduction
```

***

**So the answer is — loop fission:**

Split into two loops:

```c
// Loop 1 — sequential (RAW dependency, cannot parallelise)
result = temp[0];
for (int i = 1; i < N; i++) {
    data[i] = data[i] + data[i-1];
}

// Loop 2 — parallel (independent, use reduction)
result = temp[0];
#pragma omp parallel for reduction(+:result)
for (int i = 1; i < N; i++) {
    result = result + temp[i];
}
```

***

**Exam answer structure:**

```
1. Identify dependencies     → data[i] depends on data[i-1] → RAW
2. Identify what's safe      → result += temp[i] → independent
3. Apply loop fission        → split into two loops
4. Parallelise safe loop     → reduction(+:result)
5. Keep unsafe loop sequential
```

**write after read -> AntiDepednecy**

### Removing Antidependencies

```c
for (i = 0; i < N; i++) {
    a[i] = a[i+1] + c;   // reads a[i+1] which iteration i+1 will overwrite
}
```

**Intuition**: someone is erasing the whiteboard while you're still copying it. Fix: make a photocopy first.

```c
// Step 1: copy the data you need to read
for (i = 0; i < N; i++)
    a2[i] = a[i+1];

// Step 2: parallelise safely — reads from copy, writes to original
#pragma omp parallel for
for (i = 0; i < N; i++)
    a[i] = a2[i] + c;
```
